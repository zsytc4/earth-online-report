# 地球Online · 上线清单 + 引路员操作手册

---

## 第一部分:上线清单(你自己做,约30分钟)

### A. 上线前改3处代码(用任何文本编辑器打开 index.html)

| # | 找到这一行 | 改成 | 作用 |
|---|-----------|------|------|
| 1 | `const REVIEW_MODE=true;` | `const REVIEW_MODE=false;` | 关闭审阅模式,恢复闸门 |
| 2 | `const UNLOCK_CODE="破壳";` | 换成你定的暗号 | 通行暗号(建议每月换) |
| 3 | 无需改代码 | 把微信二维码截图命名为 `qrcode.png` | 闸门处自动显示 |

### B. GitHub 上线(免费,一次性)

1. 注册/登录 github.com → 右上角"+"→ New repository → 名字随意(如 earth-online)→ Public → Create
2. 进入仓库 → Add file → Upload files → 把 `index.html` 和 `qrcode.png` 拖进去 → Commit changes
3. 仓库 Settings → 左栏 Pages → Source 选 Deploy from a branch → Branch 选 main → Save
4. 等1-2分钟,页面顶部出现网址(形如 `你的用户名.github.io/earth-online`)——这就是产品链接
5. 以后改动:重复步骤2上传新版 index.html 覆盖即可

### C. 上线前必做的校准(半天)

拿5-10个你熟悉的人的生日(你心里知道他们"该是什么角色"的),挨个在网页上测:
- 职业方向对 → 通过
- 明显不对 → 记下生日+期望角色+实际输出,发给Claude调参数(强弱阈值/月令权重)
- 建议达标线:10个里对7个以上再上线

## 第二部分:引路员操作手册

### 引路员需要的全部工具

1. 微信(工作号)
2. 一个AI对话工具(Claude/DeepSeek/GPT,能力越强报告越好)
3. `深度报告_AI主提示词.md`(粘贴用)
4. `报告生成器.html`(双击打开,本地排版转PDF)
5. 本手册的话术和路由表

### 流程一:用户加微信(免费解锁)

用户会发来一串存档码,形如:
`EO-20000413-H12-精准·赏金猎人-卡关状态-Loop秩序`

**第1步 · 回暗号解锁(10秒内回复):**
> 收到,存档已登记 ✅
> 通行暗号:**破壳**(回到报告页输入即可继续)

**第2步 · 立刻打标签。** 存档码自带全部信息,在微信备注里记:
`角色-状态-Loop维度`,例:`赏金猎人-卡关-秩序`

**第3步 · 隔2-4小时,发一条基于标签的开场观察(不推销):**

按Loop维度选话术(可微调措辞):

| Loop维度 | 开场观察话术 |
|---------|-------------|
| 输出 | "看了你的存档——输出位的信号有点意思:配置里这是你的强项,但它现在几乎没在跑。你自己有'想说说不出、想做没动手'的感觉吗?" |
| 独立 | "你的存档里独立位的信号我留意了一下——重大决定是不是经常反复摇摆,或者最后交给了别人?" |
| 秩序 | "你存档里秩序位的信号偏弱——别人觉得正常的压力强度,你的系统是不是已经在报警了?这个不是抗压差,是配置问题。" |
| 资源 | "你的资源位信号有个特征:嗅觉在线但临门一脚容易自己拦自己。最近有没有明明看到机会、却没伸手的事?" |
| 认知 | "你存档里认知位的信号提示,有些方向你的系统好像'不让你去想'。你有没有那种一碰某类问题就自动绕开的感觉?" |

> 这条消息的目的:让用户觉得"他真的看了我的存档"。回复率决定后面一切。

### 流程二:深度报告(付费,建议定价 69-99)

用户说"要深度报告"后:

**第1步 · 确认信息+收款:**
> 好的。深度报告是按你的存档单独推演的,大概需要 [X小时/今晚] 生成。
> 需要补充两个信息:①最关心哪个方向(事业/关系/健康/财务)②出生城市。
> 费用 XX,微信转我即可,生成后PDF发你。

**第2步 · 生成(5分钟操作):**
1. 打开AI对话 → 粘贴《深度报告_AI主提示词》全文
2. 按模板发:存档码+性别+城市+最关心方向
3. AI输出全文 → 通读一遍,删掉任何鸡汤/恐吓残留(提示词已禁止,但要人工兜底)

**第3步 · 排版交付:**
1. 双击打开 报告生成器.html → 左侧粘贴报告全文,填角色名/词缀/存档码
2. 生成预览 → 打印 → 目标"另存为PDF" → **务必勾选"背景图形"**
3. 微信发送PDF

**第4步 · 交付话术(和PDF一起发,这句话是通向一对一的桥):**
> 报告生成好了,发你 📄
> 我自己先读了一遍——你第四节Loop溯源那里的信号比较少见,这种写入模式的玩家通常会在 [结合其Loop维度说一个具体场景] 上反复卡。你读到那一节的时候,看看有没有对上。

**第5步 · 用户回来讨论报告内容时 → 自然引出一对一:**

| Loop维度 | 对应技术(用户语言) | 对应服务 |
|---------|------------------|---------|
| 输出/独立/资源 | 改写底层信念代码 | 西塔/SRT 一对一(299-999) |
| 秩序/认知 | 调取原始存档 | 阿卡西阅读 一对一(299-999) |
| 报告第三节引发强烈共鸣 | 回到前置剧情 | 回溯/催眠 一对一 |
| 问题集中在家人/伴侣 | 家族系统排查 | 家排 |

约诊话术:
> 报告能告诉你loop在哪、从哪来,但卸载最深那一层(初始写入点)在意识层做不动——那一步需要人工进到存档层操作。如果你想处理,我这边可以安排一次深层调取,你看 [时间]?

### 流程三:装备(宝石)加购

- 用户对报告第六节装备清单感兴趣 → 报价(标准装备199-699,附鉴定证书)
- 用户要更高定制 → 转介合作老师的定制线(咨询+选石,老师交付,预约走你)
- **红线:所有预约、复购、售后都回到你的微信,客户关系不外流。**

### 每周固定动作

- 换一次通行暗号(改index.html里那一行,顺便当朋友圈梗发)
- 把本周报告案例匿名化,产出1-2条朋友圈(拆关卡帖:"一个爆燃·破阵法师的赛季卡点,我们怎么拆的")
- 统计:加微率/解锁率/深度报告转化率/一对一转化率,四个数字报周会
